Did They Perform to Your Expectations?


Semantic UI performed a lot better than we expected. It made designing our website so much easier and our CSS files were minimal. Semantic UI's responsive design proved to be quite good despite the criticism. Overall however, we would like a few more design features with Semantic UI. Django performed up to our Expectations, allowing us to create the back-end part of our website with ease. Django's email functionalities proved to be something quite useful for our website.



Was it Easy to Install?


Semantic UI proved to be very simple to install as we only needed to add links to the head of our HTML files for the features we used. Django was a little more complicated but it did not provide us with any issues during installation.



Was it easy to create the page(s) with it?


Semantic UI was very easy to create pages with. We utilized it for every page on this website for a consistent design. Django also proved to be quite easy to create pages with, especially with the features we used.



Was the learning curve steep compared with regular HTML/CSS/JavaScript/PHP?


The learning curve for Semantic UI was not steep at all. It had an easy syntax set-up and documentation that was accessible and easy to read. Django proved to have a much steeper learning curve. We had a difficult time picking up the framework and were not able to get into its deep features yet. That being said, we do look forward to learning more about the Django framework and how we could use it for more complex websites.



Would you have done things differently in retrospect? Did you regret your choice of frameworks?


While we do not regret our choice of frameworks, looking back, we may have done things differently if we knew about the steep learning curve of Django. We would have liked to experiment in a framework like Laravel that works in PHP as PHP is something we are much more familiar with. To do things differently, we may have focused more of our attention on Django and compare its features to PHP to truly understand how to use the framework.
